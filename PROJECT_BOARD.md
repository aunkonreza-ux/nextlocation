# Projects v2 Board Skeleton — Next Location — Lovable Import

This file contains a suggested Projects v2 board skeleton to import the Lovable "Next Location" project into this repository.

Board name
----------
Next Location — Lovable Import

Recommended fields
------------------
- Status (single-select): Backlog, To do, In progress, Blocked, Review, Done
- Area (single-select): frontend, backend, design, content, infra, chore
- Priority (single-select): P0, P1, P2, P3
- Effort (number): estimated story points or hours

Suggested view / columns
------------------------
Group the view by the Status field, with these columns:
- Backlog
- To do
- In progress
- Blocked
- Review
- Done

Suggested labels for imported issues
-----------------------------------
- project/next-location
- area/frontend | area/backend | area/design | area/content | area/infra
- priority/P0 | priority/P1 | priority/P2

Import mapping guidance
----------------------
- Map Lovable statuses to Status (Backlog, To do, In progress, Done).
- Map Lovable categories/tags to Area.
- Use Lovable due dates or estimates to populate Effort or milestones.

Import checklist
----------------
- [ ] Review Lovable project and decide which cards to import (exclude archived/duplicate items).
- [ ] Export Lovable project as CSV/JSON (recommended) or provide an export link.
- [ ] Map Lovable statuses and tags to the board fields above.
- [ ] Create Projects v2 board in the repository using this skeleton.
- [ ] Create GitHub issues for selected Lovable cards and include the Lovable card URL in each issue body.
- [ ] Add created issues to the Projects board and set fields/labels.
- [ ] Notify contributors when migration is complete.

How I can help
--------------
- I can create GitHub issues from a Lovable export (CSV/JSON) or from pasted card details.
- I can provide a script or step-by-step instructions for an automated migration if you provide a Lovable export or a PAT with necessary scopes.
- I can update this file to reflect any mapping changes you prefer.
